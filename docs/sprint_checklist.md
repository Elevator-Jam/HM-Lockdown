# Sprint Checklist - [Sprint Name/Number]

Use this checklist to ensure every feature follows the established **Agile & Agentic Engineering** process.

---

## 1. Planning & Design (Sprint Start)
- [ ] **Feature Design Doc (FDD):** Created using `feature_design_template.md`.
- [ ] **Agentic Audit:** AI agents used to research implementation paths or draft the technical architecture.
- [ ] **Sign-off:** Design, Art, and Engineering have reviewed and approved the document.
- [ ] **Tasks Divided:** Work split among Designers, Artists, Tech Artists, and Engineers.

## 2. Implementation (During Sprint)
- [ ] **Feature Branch:** Work is being done in a dedicated branch (not `Development` or `main`).
- [ ] **Pair Programming:** Collaboration sessions held (human-to-human or human-to-agent).
- [ ] **Style Compliance:** Code strictly adheres to `code_style_guide.md` (Braces, Naming, Headers).
- [ ] **Asset Integration:** Engineers or Tech Artists have integrated assets/UI or working mocks into Unity.
- [ ] **Daily Standup:** Team synchronizes on progress, plans, and blockers every day.
## 3. Integration & Validation (Near End)
- [ ] **Play Mode Test:** Feature verified in Unity Editor; no errors or logic bugs.
- [ ] **Platform Build:** Successful build made and tested on target hardware (iOS/Android/PC).
- [ ] **Regression Check:** Verified that NO new issues were introduced in existing systems.
- [ ] **Visual Evidence:** Screenshots or video recordings captured for the PR.

## 4. Review & Merge (Closing)
- [ ] **Pull Request:** Created from feature branch to `main`.
- [ ] **Peer Review:** At least 1 person has thoroughly reviewed and approved the code (LGTM). Or 2 people has glanced through the PR and given a +1 each.
- [ ] **Review Iterations:** All feedback addressed and confirmed by the reviewer.
- [ ] **Merge:** Feature branch merged into `main` after approval.

## 5. Retrospective (Post-Sprint)
- [ ] **Retro Meeting:** Team discussed what went well and what can be improved.
- [ ] **Process Refinement:** Any necessary updates made to docs/templates based on learnings.
- [ ] **Cleanup:** Merged feature branch deleted.
